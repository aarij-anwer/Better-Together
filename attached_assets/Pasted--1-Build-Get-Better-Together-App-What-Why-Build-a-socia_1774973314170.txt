#1 - Build Get Better Together App

What & Why
Build a social fitness challenge app where users create private, time-bound challenges (pushups, steps, etc.) and invite others to complete them together. Features progress tracking with smart backlog filling, streaks, and leaderboards.

Done looks like
Unauthenticated users see a minimal welcome screen with the app name, a brief tagline, and a "Sign in with Replit" button

Visiting an invite link (/join/:inviteCode) while logged out shows the challenge preview (title, activity, duration, participant count) with a prompt to sign in — after signing in, the user is redirected back to join that challenge

All app functionality (dashboard, challenges, logging, leaderboard) is behind authentication

Users can sign in with Replit Auth

Dashboard shows a lightweight "Today" summary strip (e.g. "Today: 30 pushups, 5,000 steps" or "2 of 3 challenges completed") above a list of active challenge cards

🔧 NEW / FIX: If a user has only one active challenge, they are taken directly to that challenge detail screen instead of the dashboard

Users can create a challenge with title, activity type, type (daily/total), target value, duration, and optional future start date (defaults to today)

🔧 NEW / FIX: Unit is NOT user-defined — it is derived from activity type (e.g. pushups → reps, running → km)

Challenges with future start dates appear as "not started" with logging disabled until the start date

🔧 NEW / FIX: If a challenge has not started yet:
- Logging is disabled
- Leaderboard shows participants only (no rankings yet)
- Streak is zero for all participants

An invite code is auto-generated; users can join challenges via /join/:inviteCode or by entering a code from the dashboard

Users can log progress with quick-add buttons (+10, +20, +50) or custom input — logging fills today first, then overflows to oldest incomplete days (no overfilling)

🔧 NEW / FIX: Logging behavior is deterministic:
- Always fills today's remaining target first
- Remaining amount flows to oldest incomplete days
- A day cannot exceed its target (no overfilling)

Challenge detail screen shows: today's progress with progress ring, horizontal day selector, quick-add buttons, and a compact leaderboard preview

🔧 NEW / FIX: If no progress has been logged yet, show an empty state prompting the user to log their first activity (e.g. “Start your first set 💪”)

Full leaderboard shows participants ranked by total completed with % completion as tiebreaker

Streaks are calculated as consecutive completed days (recoverable when backlog is filled), shown with streak indicator on challenge detail

Empty states guide new users toward creating or joining their first challenge

Design direction
Clean, minimal, and modern — inspired by Notion (structure) and Apple Fitness (progress visuals)

Strong spacing and typography with clear visual hierarchy

Today's progress is visually dominant on the challenge detail screen

Subtle motion: progress ring animation on log, smooth transitions

Limited color palette with one primary accent color

Calm and motivating tone — social and competitive without being aggressive

No gamification clutter (no excessive badges, loud animations, or competing elements)

Core loop: Open app > log progress > see progress > stay motivated

Out of scope
Apple Health integration
Teams (just individual participants)
Public challenge discovery
Comments/chat
Push notifications (just UI stubs)
Editing start date after creation
Charts, graphs, or historical analytics on the dashboard
